Benchmark Mode

Systematically compares baseline vs auto-N vs stability vs ensemble across one or more tickers using walk-forward evaluation.

📌 Positional Argument
ArgumentTypeDescription
tickers str+ One or more stock ticker symbols. Accepts indices (e.g. ^GSPC, ^IXIC, ^DJI) and individual stocks (e.g. AAPL, MSFT). When multiple tickers are given, each runs sequentially through the selected mode.
📅 Date Range Options
FlagTypeDefaultDescription
--start date 2020-01-01 Start of the data window. Ignored when --all-data is set.
--end date 2026-04-01 End of the data window. Ignored when --all-data is set.
--train-end date 2024-01-01 Train / test split boundary. Data before this date is used for FFT training; data from here onward is the held-out test set used for evaluation. Must be between --start and --end.
🔮 Prediction Options
FlagTypeDefaultDescription
--prediction-days int 30 Days to predict when no test set is available (i.e. --all-data mode without --train-end).
--n-components int 10 Number of dominant FFT frequency components to use for reconstruction and prediction. Overridden by --auto-components when that flag is set.
--forecast-horizon int 30 Horizon (in days) for soft projection confidence bands. Only relevant with --soft-projection.
--trend-type str log Trend component extracted before FFT and re-added after prediction.
log linear polynomial_2 polynomial_3 none
log — best for inflationary long-term series (S&P, NASDAQ).
none — pure FFT oscillation, no trend; use for short windows.
--max-cycle-ratio float 0.33 Filters out FFT components whose implied period exceeds n × ratio trading days (where n is training window length). Default of 0.33 ensures at least 3 full cycles fit inside the window, suppressing spurious very-long cycles.
🚩 Mode Flags
FlagTypeDescription
--all-data, -a bool Fetch the maximum available history (back to 2000-01-01). Ignores --start and --end. Recommended for window optimizer and ensemble modes.
--backtest bool Switch to backtest mode. Simulates a long/short strategy using FFT direction signals. Produces a trade log CSV, equity curve CSV, and prints Sharpe, drawdown, and return metrics.
--benchmark bool Compare four prediction methods (baseline, auto_n, stability, ensemble) across all provided tickers using walk-forward MAPE. Saves a plain-text report to outputs/benchmark_report.txt.
--reconstruct bool Include a signal reconstruction panel in the dashboard, showing how well the top-N FFT components reproduce the original price series.
--soft-projection bool Use the soft-projection method from reconstruction.py. Generates a smoother forecast with explicit confidence bands. Disables trend-based prediction and window/ensemble optimizations.
🧠 Adaptive & Ensemble Options

These flags run as sub-steps 3b → 3c → 3d inside the prediction pipeline and can be combined freely. They have no effect when --soft-projection is set or when --trend-type none is used.

FlagTypeDescription
--optimize-window bool Step 3b. Tests six window lengths (504 d ≈ 2yr, 756 ≈ 3yr, 1260 ≈ 5yr, 2520 ≈ 10yr, 3780 ≈ 15yr, 5040 ≈ 20yr) and picks the best via a composite score:
0.5 × MAPE + 0.3 × (100 − DirAcc) + 0.2 × (1 − Stability) × 100
The training data is then trimmed to the winning window length.
--auto-components bool Step 3c. Runs cross-validation across candidate N values (typically 3–30) and selects the N that minimises hold-out MAPE. Overrides --n-components.
--stability-weighted bool Step 3d. Computes a persistence score for each FFT cycle period by measuring how consistently it appears across rolling sub-windows. Components are ranked by amplitude × (0.5 + 0.5 × stability) instead of pure amplitude — boosting cycles that are genuinely recurring.
--ensemble bool Step 4 (replaces single prediction). Runs predictions from four windows (2yr, 3yr, 5yr, 10yr) and combines them into a single forecast. Confidence bands are derived from the inter-window spread (± 1.5 σ). Disagreement score (mean std / mean price) is printed to stdout.
--ensemble-weighting str How ensemble windows are weighted. Only relevant when --ensemble is set.
performance stability equal
performance — weight ∝ 1/MAPE on a 20-day held-out validation set (recommended).
stability — weight ∝ spectral stability of that window.
equal — simple average of all windows.
💾 Output Options
FlagTypeDefaultDescription
--output-dir str outputs Directory where all output files are written. Created automatically if it does not exist.
--save-to-db bool false Persist predictions to a SQLite database in addition to the default CSV output.
--db-path str outputs/predictions.db Path to the SQLite database file. Only used when --save-to-db is set.
📈 Backtest-Specific Options
FlagTypeDefaultDescription
--prediction-window int 252 Number of trading days of history fed into FFT at each backtest step (252 ≈ 1 year).
--hold-period int 5 Days to hold a position after entering. Shorter = more frequent trades; longer = reduced transaction noise.
--initial-capital float 10000 Starting capital in USD for the backtest simulation.
📁 Generated Output Files
FileModeDescription

{TICKER}_predictions.csv Pipeline Date, actual price, predicted price, difference, and % error for each test day.
predictions.db Pipeline (--save-to-db) SQLite database with one row per prediction. Path configurable via --db-path.

💡 Example Commands
Basic

Prediction on a specific date range

python main.py AAPL \ --start 2023-01-01 \ --end 2025-01-01 \ --train-end 2024-06-01
Trains on 2023-01-01 → 2024-06-01, evaluates on 2024-06-01 → 2025-01-01. Uses default log trend and 10 FFT components.
Basic

All available data — quickest way to get a forecast

python main.py ^GSPC --all-data --trend-type log
Fetches data from 2000-01-01. No manual date range needed. Good starting point before enabling adaptive options.
Adaptive

Auto-select best historical window (2yr – 20yr)

python main.py ^GSPC --all-data --optimize-window --trend-type log
Runs walk-forward evaluation on six window lengths and trims the training set to the winner. Prints a ranked table of all windows. Use when unsure how much history is relevant.
Adaptive

Cross-validated N components

python main.py AAPL --all-data --auto-components --trend-type log
Finds the best --n-components via cross-validation instead of relying on the default of 10. Prints CV scores for each N tried.
Adaptive

Stability-weighted component selection

python main.py AAPL --all-data --stability-weighted --trend-type log
Measures how consistently each FFT cycle appears across rolling sub-windows. Components that recur reliably get a higher effective amplitude — ephemeral noise gets down-weighted.
Ensemble

Multi-window ensemble — most robust forecast

python main.py ^GSPC --all-data \ --ensemble \ --ensemble-weighting performance \ --trend-type log
Combines four separate predictions (2yr, 3yr, 5yr, 10yr windows) weighted by each window's recent held-out MAPE. Confidence bands come from inter-window spread (± 1.5 σ).
Full Stack

All adaptive improvements combined

python main.py AAPL ^GSPC --all-data \ --optimize-window \ --auto-components \ --stability-weighted \ --ensemble \ --ensemble-weighting performance \ --trend-type log \ --save-to-db
Stacks all four adaptive sub-steps on two tickers sequentially. Steps 3b → 3c → 3d feed into the ensemble at step 4. Results are also persisted to SQLite.
Backtest

Historical backtest with custom hold period

python main.py AAPL \ --backtest \ --start 2020-01-01 \ --end 2024-12-31 \ --hold-period 10 \ --initial-capital 50000 \ --trend-type log
Simulates trading AAPL from 2020 to 2024 with 10-day hold periods and $50k capital. Outputs Sharpe ratio, max drawdown, total return, and a full trade log CSV.
Benchmark

Systematic method comparison

python main.py ^GSPC AAPL MSFT \ --benchmark \ --start 2015-01-01 \ --trend-type log \ --n-components 10
Runs all four methods (baseline, auto_n, stability, ensemble) on three tickers over a 10-year span. Prints a ranked comparison table and saves it to outputs/benchmark_report.txt.
Output

Save to SQLite with custom path

python main.py MSFT \ --all-data \ --save-to-db \ --db-path data/my_predictions.db \ --output-dir data/results
Writes the dashboard HTML and CSV to data/results/ and the predictions into a named SQLite file instead of the default location.
Reconstruction

Show how well FFT fits the training signal

python main.py AAPL \ --all-data \ --reconstruct \ --n-components 15 \ --trend-type log
Adds a reconstruction panel to the dashboard using the top 15 components. Useful for diagnosing underfitting (too few components) or overfitting (too many).
Soft Projection

Smooth forecast with confidence bands

python main.py ^IXIC \ --all-data \ --soft-projection \ --forecast-horizon 60 \ --n-components 8
Uses the soft-projection algorithm for a smoother, confidence-band-equipped forecast. Note: disables window optimizer and ensemble options.
